## Bare Metal Quick Start

➕ Bencher Plus

This guide walks you through running your first benchmark on a Bare Metal Runner.
We will use the `bencher` CLI Docker image as the benchmark image
and [`bencher noise`][bencher noise] as the example benchmark command.
No custom image is needed.

### Run a Benchmark on Bare Metal

The [`bencher noise`][bencher noise] subcommand measures environment noise on the Runner hardware.
By passing `--format json`, it outputs results in [Bencher Metric Format (BMF)][bmf],
which the `magic` adapter auto-detects.

We will use the `intel-v1` [Spec][spec page] to target an Intel-based Bare Metal Runner.
See the [Spec page][spec page] for a full list of available Specs and their hardware details.

Run the following command:

```
bencher run --image <bencher-cli-image> --spec intel-v1 bencher noise --format json
```

Everything after the `bencher run` options is the benchmark command that executes inside the container.
In this case, the benchmark command is `bencher noise --format json`.

The `--image` option sets the OCI image for remote execution,
and the `--spec` option selects the hardware Spec for the Runner.

### View Your Results

After the run completes, view your results in the Bencher Console.
The output of `bencher run` includes a `View results` link that you can click or copy into your browser.

### Next Steps

To run benchmarks for your own project on Bare Metal,
you will need to create a custom OCI image that contains your benchmark code and dependencies.
See the [Image page][image page] for instructions on how to build and push your own image.

For more details, see the [bencher run documentation][bencher run]
and the [Quick Start tutorial][quick start] for getting started with Bencher.

### `plus.registry`

이 섹션은 OCI(Open Container Initiative) 레지스트리 스토리지 구성을 지정합니다.
Bencher는 OCI 호환 컨테이너/아티팩트 레지스트리로 작동할 수 있습니다.

이 섹션이 구성되지 않은 경우, Bencher는 데이터베이스 파일 옆의
`oci` 디렉토리에 로컬 파일시스템 스토리지를 사용합니다.
이는 개발 및 단일 인스턴스 배포에 적합합니다.

여러 인스턴스가 있는 프로덕션 배포 또는 내구성을 위해서는
S3 기반 스토리지를 구성하세요.

하나의 스토리지 `service`가 있습니다:
- `aws_s3`: AWS S3에 OCI 아티팩트 저장
    - `access_key_id`: AWS 액세스 키 ID
    - `secret_access_key`: AWS 시크릿 액세스 키
    - `access_point`: 선택적 경로 접두사가 있는 S3 액세스 포인트 ARN.
      형식: `arn:aws:s3:<region>:<account-id>:accesspoint/<access-point-name>[/<path-prefix>]`

### `plus.rate_limiting`

本节指定 API 服务器的速率限制。
整节内容是可选的。
如果未指定，将使用列出的默认值。

为防止创建[`unclaimed`项目][unclaimed projects]，请将`unclaimed_limit`设置为`0`。

|          名称          |      示例      |    默认值    | 是否必需 |                                                 描述                                                  |
| :--------------------: | :----------: | :----------: | :------: | :---------------------------------------------------------------------------------------------------: |
|         window         |    86400     |    86400     |    否    |                            指定以秒为单位的速率限制窗口。默认设置为一天。                             |
|    unclaimed_limit     |     255      |     255      |    否    | 控制[`unclaimed`项目][unclaimed projects]的速率限制。为了防止创建`unclaimed`项目，请将此值设置为`0`。 |
|     claimed_limit      |    65536     |    65536     |    否    |                                     控制`claimed`项目的速率限制。                                     |
|     oci_bandwidth      |  \{ ... \}   |     ---      |    否    |                              指定每个组织层级的每日OCI注册表带宽限制。                                 |
| oci_bandwidth.unclaimed | 1073741824  | 1073741824   |    否    |                          未认领组织（0个成员）每天的最大字节数。默认为1 GiB。                          |
|   oci_bandwidth.free   | 10737418240  | 10737418240  |    否    |                          免费（已认领，无付费计划）组织每天的最大字节数。默认为10 GiB。                |
|   oci_bandwidth.plus   | 107374182400 | 107374182400 |    否    |                          Plus（Team/Enterprise）组织每天的最大字节数。默认为100 GiB。                 |

[unclaimed projects]: /zh/docs/how-to/claim/#unclaimed-projects

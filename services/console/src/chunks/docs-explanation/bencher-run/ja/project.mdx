### `--project <PROJECT>`

<br />

任意: `--project` オプションまたは環境変数 `BENCHER_PROJECT` のいずれかに、Project のスラッグまたは UUID を設定できます。
指定された値がスラッグで、まだ Project が存在しない場合は自動的に作成されます。
ただし、指定された値が UUID の場合は、Project が既に存在している必要があります。
両方が指定されている場合は、`--project` オプションが環境変数 `BENCHER_PROJECT` より優先されます。
どちらも指定されていない場合、以下に基づいて Project スラッグが自動生成されます:

1. 利用可能であれば、その `git` リポジトリの親ディレクトリ名。
2. 利用可能であれば、その `git` リポジトリの初回コミットの 7 桁の 16 進数短縮ハッシュ。
3. サポートされている OS 上のローカルマシンの 13 桁の英数字フィンガープリント。

例えば、生成される Project スラッグは次のようになることがあります: `project-abc4567-wxyz123456789`

スラッグが指定されている場合でも自動生成された場合でも、新しい Project は必ず Organization に所属します。
ユーザーが認証されている場合、そのユーザーの個人 Organization の下に Project が追加されます。
ユーザーが認証されていない場合、Project は新しい `unclaimed` Organization の下に作成されます。
この Organization は、Project の公開 Perf ページから `claimed` するか、後続の `bencher run` 実行時に認証した状態で Project スラッグを使用することで取得できます。

> 🐰 重要: `CI` 環境変数が `true` に設定されている場合、
> プロジェクトを指定するか、
> [`--ci-on-the-fly` フラグ][ci on the fly] を設定する必要があります。

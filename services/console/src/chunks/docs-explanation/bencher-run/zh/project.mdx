### `--project <PROJECT>`

<br />

可选：可以将 `--project` 选项或 `BENCHER_PROJECT` 环境变量设置为项目的 slug 或 UUID。
如果指定的值是 slug 且该项目不存在，则会为你创建该项目。
但如果指定的是 UUID，则该项目必须已存在。
如果两者都指定，则 `--project` 选项优先于 `BENCHER_PROJECT` 环境变量。
如果两者都未指定，则会根据以下信息为你生成一个项目 slug：

1. 如果可用，则使用 `git` 仓库的父目录名。
2. 如果可用，则使用 `git` 仓库初始提交的 7 位十六进制短哈希。
3. 对于支持的操作系统，使用本地机器的 13 位字母数字指纹。

例如，生成的项目 slug 可能看起来像：`project-abc4567-wxyz123456789`

当为你创建新项目时，无论 slug 是指定的还是生成的，该项目必须属于某个组织。
如果用户已认证，则该项目会被添加到该用户的个人组织下。
如果用户未认证，则该项目会在一个新的 `unclaimed` 组织下创建。
该组织随后可以通过项目的公开 Perf 页面进行 `claimed`，或在后续已认证的 `bencher run` 调用中使用项目 slug 来认领。

> 🐰 重要：如果 `CI` 环境变量被设置为 `true`，
> 那么你需要要么指定项目，
> 要么设置[`--ci-on-the-fly` 标志][即时 CI]。

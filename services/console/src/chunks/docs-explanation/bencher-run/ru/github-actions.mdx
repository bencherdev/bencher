### `--github-actions <GITHUB_TOKEN>`

<br />

Необязательно: Установите токен аутентификации API GitHub. Самый удобный способ сделать это - использовать [переменную окружения `GITHUB_TOKEN` в GitHub Actions][github token] (например, `--github-actions ${{ secrets.GITHUB_TOKEN }}`). Когда эта опция установлена и `bencher run` используется в GitHub Actions, результаты будут добавлены к head-коммиту как проверка GitHub с именем `Bencher Report` (или `Bencher Report (<ID>)`, если установлен `--ci-id`). Проверка GitHub завершится неудачей, если сгенерировано предупреждение, поэтому ее можно использовать как обязательную проверку статуса в защите веток. Проверка GitHub создается со статусом `in_progress` до запуска бенчмарков и завершается с результатами после их окончания. Если `bencher run` завершается с ошибкой до публикации результатов, проверка GitHub помечается как неудавшаяся. Это требует, чтобы у токена были [полномочия `checks` с разрешениями на `write`][github token permissions]. Если проверку GitHub не удается создать, `bencher run` запишет предупреждение в журнал и продолжит работу. Когда `bencher run` используется как часть запроса на вытягивание, результаты также будут добавлены в запрос как комментарий. Это требует, чтобы у токена были [полномочия `pull-requests` с разрешениями на `write`][github token permissions]. Результаты также всегда будут добавлены в сводку задания.

[github token]: https://docs.github.com/ru/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/ru/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 Если вы запускаете внутри контейнера Docker в GitHub Action, вам нужно будет передать следующие переменные окружения и смонтировать путь, указанный в `GITHUB_EVENT_PATH`:
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
> - `GITHUB_API_URL` (GitHub Enterprise Server только)

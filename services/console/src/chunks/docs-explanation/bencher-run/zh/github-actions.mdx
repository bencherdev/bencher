### `--github-actions <GITHUB_TOKEN>`

<br />

可选：设置 GitHub API 身份认证令牌。最方便的方法是使用 [GitHub Actions `GITHUB_TOKEN` 环境变量][github token]（即 `--github-actions ${{ secrets.GITHUB_TOKEN }}`）。当这个选项设置好并且 `bencher run` 在 GitHub Actions 中使用时，结果将作为名为 `Bencher Report` 的 GitHub 检查（如果设置了 `--ci-id`，则为 `Bencher Report (<ID>)`）添加到头提交中。如果生成了警报，GitHub 检查将失败，因此它可以用作分支保护中的必需状态检查。GitHub 检查会在基准测试开始之前以 `in_progress` 状态创建，并在基准测试结束后随结果一起完成。如果 `bencher run` 在结果发布之前因错误退出，GitHub 检查将被标记为失败。这要求令牌具有 [具备 `write` 权限的 `checks` 范围][github token permissions]。如果无法创建 GitHub 检查，`bencher run` 将记录一条警告并继续。当 `bencher run` 作为拉取请求的一部分使用时，结果还将作为一条评论添加到拉取请求中。这要求令牌具有 [具备 `write` 权限的 `pull-requests` 范围][github token permissions]。结果也将始终被添加到作业摘要中。

[github token]: https://docs.github.com/zh/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/zh/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 如果您在 GitHub Action 中的 Docker 容器内运行，您将需要传入以下环境变量并挂载由 `GITHUB_EVENT_PATH` 指定的路径：
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
> - `GITHUB_API_URL` (GitHub Enterprise Server 专用)

## Spec

Spec — это спецификация оборудования, описывающая ресурсы, доступные [Runner][runner]:

1. Операционная система (ОС)
   1. Linux
   2. macOS
   3. Windows
2. Архитектура процессора
   1. x86_64
   2. ARM64
3. Sandbox
   1. Firecracker microVM
   2. Без Sandbox
4. Количество ядер процессора
5. Объём памяти (RAM)
6. Размер диска
7. Доступ к сети

Spec имеют область действия либо на уровне сервера, либо на уровне организации,
поэтому один и тот же Spec может использоваться несколькими [Testbed][testbed] на одном сервере или в одной организации соответственно.

Порядок приоритета при выборе Spec при использовании <code><a href="/docs/explanation/bencher-run/">bencher run</a></code> следующий:
1. Опция `--spec`
2. Spec, назначенный Testbed
3. Spec по умолчанию на уровне организации
4. Spec по умолчанию на уровне сервера

Выбранный Spec будет затем назначен текущему [Testbed][testbed].

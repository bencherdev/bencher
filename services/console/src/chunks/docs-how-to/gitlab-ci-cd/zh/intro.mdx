根据您的使用案例，您可以在 GitLab CI/CD 中为您的以下项设置[持续基准测试][continuous benchmarking]：

- [目标分支](#target-branch)
- [合并请求](#merge-requests)

确保在继续之前已创建项目 API 密钥并将其设置为名为 `BENCHER_API_KEY` 的屏蔽变量！导航到 `Your Repo -> Settings -> CI/CD -> Variables -> Expand -> Add variable`。变量键应为 `BENCHER_API_KEY`，变量值应为您的项目 API 密钥。勾选 `Protect variable` 和 `Mask variable` 这两个选项。

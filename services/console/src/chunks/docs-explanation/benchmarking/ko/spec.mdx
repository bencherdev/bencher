#### Spec
Spec은 테스트베드에서 사용할 수 있는 리소스를 설명하는 하드웨어 사양입니다.
CPU 아키텍처, CPU 수, 메모리 크기, 디스크 크기, 그리고 환경의 네트워크 접근 여부가 포함됩니다.
Spec은 서버 범위이므로, 동일한 Spec을 여러 테스트베드에서 공유할 수 있습니다.

실행이 Bare Metal Job을 생성하면, Spec은
하드웨어 리소스와 기본 Testbed 이름을 모두 결정합니다:

| `--testbed` 제공 | Job 요청 | Testbed 이름 소스          |
|-----------------|---------|---------------------------|
| 아니오           | 아니오   | 컨텍스트 OS               |
| 예              | 아니오   | 사용자의 명시적 값         |
| 아니오           | 예      | 해결된 Spec의 이름 (Spec이 Testbed에 할당됨) |
| 예              | 예      | 사용자의 명시적 값 (Spec이 Testbed에 할당됨) |

### 테스트베드
테스트베드는 리포트를 실행할 때 사용되는 테스트 환경의 이름입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 하위 명령어를 사용할 때, 호스트 운영 체제를 기준으로 `Linux`, `macOS`, 또는 `Windows`가 기본 테스트베드로 사용됩니다.
만약 `bencher` CLI가 다른 운영 체제용으로 컴파일되었다면, `localhost`가 사용됩니다.

테스트베드에는 선택적으로 Spec을 할당할 수 있습니다.
Spec은 테스트베드가 실행되는 하드웨어를 설명합니다.
리포트가 생성되면, 해당 시점의 테스트베드의 현재 Spec이 리포트에 기록됩니다.
이를 통해 테스트베드의 Spec이 시간이 지남에 따라 변경되더라도
벤치마크 결과가 기반 하드웨어와 어떻게 관련되는지 추적할 수 있습니다.

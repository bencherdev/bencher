## Le Benchmarking Continu vs Gestion de la Performance des Applications (APM)

La Gestion de la Performance des Applications (APM) est un outil essentiel pour les services logiciels modernes.
Cependant, l'APM est conçu pour être utilisé en production.
Lorsqu'une régression de performance est détectée, elle a déjà un impact sur vos clients.

> La plupart des défauts finissent par coûter plus cher que ce qu'il aurait coûté de les prévenir.
> Les défauts sont coûteux lorsqu'ils se produisent, à la fois par les coûts directs de correction
> et par les coûts indirects dus aux relations endommagées, aux pertes d'affaires et au temps de développement perdu.
>
> — Kent Beck, Extreme Programming Explained

Bencher offre plusieurs fonctionnalités que les outils APM ne peuvent pas offrir :
- Détecter les régressions de performance _avant_ qu'elles ne soient fusionnées
- Inclure les changements et impacts sur la performance dans la revue de code
- Pas de surcharge dans les environnements de production
- Efficace pour les déploiements sur site
- Aucune modification du code source de production

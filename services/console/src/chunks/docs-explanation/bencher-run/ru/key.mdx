### `--key <KEY>`

<br />

Необязательно: Либо опция `--key`, либо переменная окружения `BENCHER_API_KEY`
может быть установлена в действительный API ключ.
Если оба заданы, опция `--key` имеет приоритет над переменной окружения `BENCHER_API_KEY`.
Существует два вида API ключей:

- Ключ API пользователя (`bencher_user_*`) привязан к пользователю и выполняет аутентификацию от имени этого пользователя.
  [Нажмите здесь, чтобы создать ключ API пользователя](/console/users/keys/add?back=L3J1L2RvY3MvZXhwbGFuYXRpb24vYmVuY2hlci1ydW4vIy0ta2V5LWtleQ==).
- Ключ API проекта (`bencher_run_*`) ограничен одним проектом
  и может использоваться только для `bencher run` и других неразрушающих команд.
  Ключ API проекта нельзя использовать для создания нового Проекта или присвоения существующего Проекта,
  поэтому требуется, чтобы [опция `--project`](#--project-project) была установлена в существующий Проект.

Если не заданы ни опция `--key`, ни [устаревшая опция `--token`](#--token-token),
то Проект должен быть `unclaimed`.
Конфликтует с [устаревшей опцией `--token`](#--token-token).

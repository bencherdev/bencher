## 벤치마크
벤치마크는 이름이 지정된 성능 회귀 테스트입니다.
성능 회귀 테스트가 Bencher에 처음 추가되는 경우 벤치마크가 자동으로 생성됩니다.
그렇지 않으면 성능 회귀 테스트의 이름이 벤치마크의 고유 식별자로 사용됩니다.

성능 회귀 테스트의 이름을 변경할 때는 주의하세요.
Bencher에서 벤치마크 이름을 수동으로 변경하여 새 이름과 일치시켜야 합니다.
그렇지 않으면 이름이 변경된 성능 회귀 테스트는 새로운 벤치마크로 간주됩니다.
이 주의사항은 일부 성능 회귀 테스트를 _이동_할 때에도 동일하게 적용됩니다.
벤치마크 하니스에 따라 성능 회귀 테스트의 경로가 이름의 일부가 될 수 있습니다.

위의 주의사항의 유일한 예외는 벤치마크를 무시하는 경우입니다.
자세한 내용은 [알림 억제][suppressing alerts]를 참조하세요.

[Bencher 메트릭 형식(BMF)][bmf]을 사용하는 경우 이름, 슬러그(slug) 또는 UUID로 벤치마크를 식별할 수 있습니다.

[suppressing alerts]: /ko/docs/explanation/thresholds/#suppressing-alerts
[bmf]: /ko/docs/reference/bencher-metric-format/

## Spec

Spec 是一个硬件规格，描述了 [Runner][runner] 可用的资源：

1. 操作系统 (OS)
   1. Linux
   2. macOS
   3. Windows
2. CPU 架构
   1. x86_64
   2. ARM64
3. Sandbox
   1. Firecracker microVM
   2. 无 Sandbox
4. CPU 数量
5. 内存大小 (RAM)
6. 磁盘大小
7. 网络访问

Spec 的作用域为服务器级别或组织级别，
因此同一个 Spec 可以分别在同一服务器上或同一组织内的多个 [Testbeds][testbed] 之间共享。

使用 <code><a href="/zh/docs/explanation/bencher-run/">bencher run</a></code> 时，选择 Spec 的优先级顺序如下：
1. `--spec` 选项
2. 分配给 Testbed 的 Spec
3. 默认的组织级别 Spec
4. 默认的服务器级别 Spec

选定的 Spec 将被分配给当前的 [Testbed][testbed]。

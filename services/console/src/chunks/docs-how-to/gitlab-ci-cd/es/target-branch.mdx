## Rama Objetivo

Una piedra angular de [Benchmarking Continuo EstadĂ­stico][statistical continuous benchmarking]
es tener una lĂ­nea base histĂłrica para su rama objetivo.
Esta lĂ­nea base histĂłrica puede utilizarse para detectar regresiones de rendimiento en Solicitudes de FusiĂłn.

1. Cree un archivo de CI/CD de GitLab.
   (ej: `.gitlab-ci.yml`)
2. Cree un trabajo de CI/CD de GitLab.
   (ej: `benchmark_target_branch`)
3. Ejecute `if` el pipeline fue disparado por un `push` a la rama `main`.
   Vea la [documentaciĂłn de `rules` de CI/CD de GitLab][gitlab ci cd rules]
   y la [documentaciĂłn de variables predefinidas de CI/CD de GitLab][gitlab ci cd redefined variables] para una visiĂłn completa.
   (ej: `rules: if: ...`)
4. Configure la `image` en la que se ejecutarĂˇ el trabajo.
   Vea la [documentaciĂłn de `image` de CI/CD de GitLab][gitlab ci cd image] para una visiĂłn completa.
   (ej: `image: debian:bullseye`)
5. Instale la CLI de Bencher usando [el script de conveniencia][install bencher cli].
   (ej: `before_script: ...`)
6. Use el subcomando de CLI <code><a href="/es/docs/explanation/bencher-run/">bencher run</a></code>
   para ejecutar sus benchmarks de la rama `main`.
   Vea [el subcomando `bencher run` de la CLI][bencher run] para una visiĂłn completa.
   (ej: `bencher run`)
7. Configure la opciĂłn `--project` al slug del proyecto.
   Vea [los documentos de `--project`][project option] para mĂˇs detalles.
   (ej: `--project project-abc4567-wxyz123456789`)
8. Configure la opciĂłn `--token` a la variable de entorno enmascarada `BENCHER_API_TOKEN`.
   Vea [los documentos de `--token`][token option] para mĂˇs detalles.
   (ej: `--token "$BENCHER_API_TOKEN"`)
9. Configure la opciĂłn `--branch` al nombre de la rama.
   Vea [los documentos de `--branch`][branch option] para una visiĂłn completa.
   (ej: `--branch main`)
10. Configure la opciĂłn `--testbed` al nombre del Testbed.
    Esto probablemente deba coincidir con la mĂˇquina seleccionada en `image`.
    Vea [los documentos de `--testbed`][testbed option] para mĂˇs detalles.
    (ej: `--testbed debian:bullseye`)
11. Configure el Umbral para la rama `main`, el Testbed `debian:bullseye` y la Medida `latency`:
    1. Configure la opciĂłn `--threshold-measure` a la Medida `latency` incorporada que se genera por <code><a href="/es/docs/reference/bencher-metric-format/#bencher-mock">bencher mock</a></code>.
    Vea [los documentos `--threshold-measure`][threshold measure option] para mĂˇs detalles.
    (ej: `--threshold-measure latency`)
    2. Configure la opciĂłn `--threshold-test` a una prueba t de Student (`t_test`).
    Vea [los documentos de `--threshold-test`][threshold test option] para una visiĂłn completa.
    (ej: `--threshold-test t_test`)
    3. Configure la opciĂłn `--threshold-max-sample-size` al tamaĂ±o de muestra mĂˇximo de `64`.
    Vea [los documentos de `--threshold-max-sample-size`][threshold max sample size] para mĂˇs detalles.
    (ej: `--threshold-max-sample-size 64`)
    4. Configure la opciĂłn `--threshold-upper-boundary` al LĂ­mite Superior de `0.99`.
    Vea [los documentos de `--threshold-upper-boundary`][threshold upper boundary] para mĂˇs detalles.
    (ej: `--threshold-upper-boundary 0.99`)
    5. Configure la bandera `--thresholds-reset` para que solo el Umbral especificado estĂ© activo.
    Vea [los documentos de `--thresholds-reset`][thresholds reset] para una visiĂłn completa.
    (ej: `--thresholds-reset`)
12. Configure la bandera `--error-on-alert` para que el comando falle si se genera una Alerta.
    Vea [los documentos de `--error-on-alert`][alert err] para una visiĂłn completa.
    (ej: `--error-on-alert`)
13. Configure la opciĂłn `--adapter` a [Bencher Metric Format JSON (`json`)][bmf] que se genera por <code><a href="/es/docs/reference/bencher-metric-format/#bencher-mock">bencher mock</a></code>.
    Vea [adaptadores de benchmark harness][adapter json] para una visiĂłn completa.
    (ej: `--adapter json`)
14. Especifique los argumentos del comando de benchmark.
    Vea [comando de benchmark][command argument] para una visiĂłn completa.
    (ej: `bencher mock`)

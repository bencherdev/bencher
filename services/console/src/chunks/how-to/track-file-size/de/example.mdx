Wenn Sie ein Skript unter `./my_build_script.sh` hätten, das Ihr Binär an dem Pfad `./path/to/my_binary` erstellt,
dann könnten Sie die Größe dieser Binärdatei mit
<code><a href="/de/docs/explanation/bencher-run/">bencher run</a></code>
und [dem `json` Adapter][json adapter] verfolgen.
Das würde sowohl mit [einem Benchmark-Befehl][bencher run] als auch ohne einen solchen funktionieren.

In diesem Beispiel ist der Schlüssel `my_binary` der Dateiname der Binärdatei.
Er wird als Name des [Benchmarks][benchmark] verwendet.
Das `my_binary` Objekt enthält den Schlüssel `file-size`.
`file-size` ist der Slug für die integrierte Dateigröße [Messung][measure].
Die Dateigrößenmessung wird nicht standardmäßig für alle Projekte erstellt.
Jedoch, wenn Sie die Dateigrößenmessung verwenden,
wird sie automatisch für Ihr Projekt erstellt.
Das Dateigrößenmessungsobjekt enthält eine [Metrik][metric]
mit dem Dateigrößen-`value` in Bytes, `42.0`.

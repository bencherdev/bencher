## Démarrer le Runner

Avec un Runner créé et au moins un Spec assigné,
démarrez le binaire `runner` sur votre matériel avec [`runner up`][runner up].
Fournissez l'hôte du serveur API, l'UUID ou le slug du Runner, et la clé du Runner,
soit en tant que drapeaux, soit en tant que variables d'environnement :

```sh
runner up \
  --host https://api.bencher.dev \
  --runner my-runner \
  --key bencher_runner_...
```

```sh
export BENCHER_HOST=https://api.bencher.dev
export BENCHER_RUNNER=my-runner
export BENCHER_RUNNER_KEY=bencher_runner_...
runner up
```

Le binaire `runner` ouvre une unique [connexion WebSocket][runner protocol] vers le serveur API,
interroge les Jobs correspondant à ses Specs, exécute chacun d'eux, et renvoie les résultats.
La connexion reste ouverte d'un Job à l'autre,
et le binaire [se met à jour lui-même][runner up] entre les Jobs lorsqu'une nouvelle version est disponible.

Par défaut, un Runner rejette tout Job dont le Spec n'a pas de [Sandbox][sandbox].
Pour permettre à un Runner d'exécuter directement des Jobs sans sandbox sur l'hôte,
démarrez-le avec `--danger-allow-no-sandbox`.

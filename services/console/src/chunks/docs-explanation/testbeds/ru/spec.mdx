### `--spec <SPEC>`

<br />

Укажите слаг или UUID для Spec.
Spec — это аппаратная спецификация, описывающая ресурсы, доступные [Runner][runner]:

1. Операционная система (ОС)
   1. Linux
   2. macOS
   3. Windows
2. Архитектура процессора
   1. x86_64
   2. ARM64
3. Sandbox
   1. Firecracker microVM
   2. Без Sandbox
4. Количество ядер процессора
5. Объём памяти (RAM)
6. Размер диска
7. Доступ к сети

В настоящее время в [Bencher Cloud][bencher cloud] доступны следующие Spec:

|   Имя    |    Slug    |  ОС   | Архитектура  |       Sandbox       | CPU |  Память  |   Диск    |  Сеть  |
| :------: | :--------: | :---: | :----------: | :-----------------: | :-: | :------: | :-------: | :----: |
| Intel v1 | `intel-v1` | Linux |    x86_64    | Firecracker microVM |  4  | 48.0 GiB | 128.0 GiB |  Нет   |

Spec бывают либо на уровне сервера, либо на уровне организации,
поэтому одну и ту же Spec можно использовать совместно на нескольких испытательных платформах одного сервера или в пределах одной организации соответственно.

Порядок приоритета при выборе Spec во время использования <code><a href="/ru/docs/explanation/bencher-run/">bencher run</a></code> следующий:
1. Опция `--spec`
2. Spec, назначенная испытательной платформе
3. Spec по умолчанию на уровне организации
4. Spec по умолчанию на уровне сервера

Выбранная Spec будет назначена текущей [испытательной платформе][testbed].
Для этой опции требуется, чтобы была установлена [опция `--image`][image].

Depending on your use case, you can set up [Continuous Benchmarking][continuous benchmarking] in GitHub Actions for your:

- [Base Branch][base branch]
- [Pull Requests][pull requests]
- [Pull Requests from Forks][fork pull requests]

Assurez-vous d'avoir créé une clé API de projet
et de l'avoir définie comme un secret de **Référentiel** nommé `BENCHER_API_KEY` avant de continuer !
Allez à `Votre Répo -> Paramètres -> Secrets et variables -> Actions -> Nouveau secret de référentiel`.
Nommez le secret `BENCHER_API_KEY` et définissez la valeur secrète sur votre clé API de projet.

Dans GitHub Actions,
[les secrets ne sont pas transmis au runner lorsqu'un workflow est déclenché depuis un dépôt forké][github actions secrets].
Par conséquent, vous devrez utiliser une branche du même dépôt
lors de l'ajout de l'un des workflows ci-dessous à votre dépôt avec une pull request.
Si vous essayez d'ajouter Bencher avec une pull request à partir d'un fork,
alors le secret `BENCHER_API_KEY` ne sera pas disponible.
`${{ secrets.BENCHER_API_KEY }}` sera une chaîne vide.

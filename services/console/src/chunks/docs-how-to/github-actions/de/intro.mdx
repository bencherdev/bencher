Je nach Anwendungsfall können Sie [Continuous Benchmarking][continuous benchmarking] in GitHub Actions für Ihre:

- [Hauptzweig][base branch]
- [Pull Requests][pull requests]
- [Pull Requests aus Forks][fork pull requests]

Stellen Sie sicher, dass Sie einen Projekt-API-Schlüssel erstellt haben und ihn als **Repository**-Geheimnis mit dem Namen `BENCHER_API_KEY` festgelegt haben, bevor Sie fortfahren!
Navigieren Sie zu `Ihr Repo -> Einstellungen -> Geheimnisse und Variablen -> Aktionen -> Neues Repository-Geheimnis`.
Benennen Sie das Geheimnis `BENCHER_API_KEY` und setzen Sie den Geheimniswert auf Ihren Projekt-API-Schlüssel.

In GitHub Actions werden
[Geheimnisse nicht an den Runner übergeben, wenn ein Workflow von einem geforkten Repository ausgelöst wird][github actions secrets].
Daher müssen Sie einen Branch aus demselben Repository verwenden,
wenn Sie eines der untenstehenden Workflows mit einem Pull Request zu Ihrem Repository hinzufügen.
Wenn Sie versuchen, Bencher mit einem Pull Request von einem Fork hinzuzufügen,
dann wird das `BENCHER_API_KEY`-Geheimnis nicht verfügbar sein.
`${{ secrets.BENCHER_API_KEY }}` wird eine leere Zeichenfolge sein.

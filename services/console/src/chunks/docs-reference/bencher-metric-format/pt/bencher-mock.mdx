## `bencher mock`

O subcomando CLI `bencher mock` é usado para gerar dados JSON simulados no formato Bencher Metric Format (BMF).

### `--seed <SEED>`

A semente para o gerador de números pseudoaleatórios.
Se não especificada, uma semente aleatória será usada.

### `--count <COUNT>`

O número de [Benchmarks][benchmark] simulados a serem gerados.

[benchmark]: /pt/docs/explanation/benchmarking/#benchmark

### `--measure <MEASURE>`

A [Measure][measure] a ser usada para cada [Benchmark][benchmark].
Várias Measures podem ser especificadas usando a opção `--measure` várias vezes.
O padrão é a Measure integrada `latency`.

[measure]: /pt/docs/explanation/benchmarking/#measure
[benchmark]: /pt/docs/explanation/benchmarking/#benchmark

### `--pow <POW>`

A potência de 10 a ser usada para as mock [Metrics][metric].
O padrão é `1`.

[metric]: /pt/docs/explanation/benchmarking/#metric

### `--fail`

Falhar durante a execução.

### `--flaky`

Falhar intermitentemente durante a execução.

### `--help`

Exibe a mensagem de ajuda.

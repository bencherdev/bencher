## Crear un Spec

Un [Spec][spec] describe el hardware que proporciona un Runner:
sistema operativo, arquitectura de CPU, tipo de [Sandbox][sandbox], número de CPU, memoria, disco y acceso a la red.
Un Job declara el Spec que necesita, y un Runner solo reclama Jobs para los Specs que soporta.

Crea un Spec con la [CLI `bencher`][bencher spec cli],
la [API REST de Specs][specs api],
o la Bencher Console:

```sh
bencher spec create \
  --host https://api.bencher.example.com \
  --name "Intel v1" \
  --os linux \
  --architecture x86_64 \
  --sandbox firecracker \
  --cpu 4 \
  --memory 51539607552 \
  --disk 137438953472
```

Omite `--sandbox` para definir un Spec sin sandbox,
y pasa `--network` si los Jobs en este Spec tienen permitido el acceso a la red.
La memoria y el disco se especifican en bytes.
Para más sobre los Specs y cómo un [Testbed][testbeds] registra el Spec usado para cada Report,
consulta [Testbeds & Specs][testbeds].

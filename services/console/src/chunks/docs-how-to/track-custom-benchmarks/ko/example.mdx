이 예제에서 키 `benchmark_name`은 [벤치마크][benchmark]의 이름이 됩니다.
벤치마크는 이름, 슬러그(slug), 또는 UUID로 식별할 수 있습니다.
`benchmark_name` 객체는 여러 [측정 항목][measure]의 이름, 슬러그, 또는 UUID를 키로 가질 수 있습니다.
지정된 값이 이름이나 슬러그이고 해당 [측정 항목][measure]이 아직 존재하지 않으면 자동으로 생성됩니다.
반면, 지정된 값이 UUID인 경우에는 해당 [측정 항목][measure]이 이미 존재해야 합니다.
이 예제에서 `latency`는 내장된 Latency Measure(지연 측정)의 슬러그입니다.
각 프로젝트는 기본적으로 Latency(즉 `latency`)와 Throughput(즉 `throughput`) Measure를 가지며,
각각 `nanosecond (ns)`와 `operations / second (ops/s)` 단위로 측정됩니다.
[측정 항목][measure] 객체는 최대 세 가지 값(`value`, `lower_value`, `upper_value`)을 가진 [지표][metric]을 포함합니다.
`lower_value`와 `upper_value` 값은 선택 사항입니다.

이 예제에서 `latency` [측정 항목][measure] 객체는 다음 값을 포함합니다:

- `value`가 `88.0`입니다.
- `lower_value`가 `87.42`입니다.
- `upper_value`가 `88.88`입니다.

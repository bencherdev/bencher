### `--ci-id <ID>`

<br />

Opcional: ID personalizado para publicar resultados no CI.  
Por padrão, o Bencher irá automaticamente segmentar os resultados pela combinação de: Projeto, Branch, Ambiente de Teste e [Adaptador](/pt/docs/explanation/adapters/).  
Definir um ID personalizado é útil quando o Bencher está sendo executado várias vezes no mesmo fluxo de trabalho de CI para a mesma combinação de Projeto, Branch, Ambiente de Teste e Adaptador.  
O ID também é anexado ao nome da Verificação do GitHub (ou seja, `Bencher Report (<ID>)`),  
de modo que cada invocação recebe um nome estável que pode ser usado como uma verificação de status obrigatória separada.  
Requer: `--github-actions`

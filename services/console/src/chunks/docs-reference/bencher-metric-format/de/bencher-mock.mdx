## `bencher mock`

Der CLI-Unterbefehl `bencher mock` wird verwendet, um simulierte Bencher Metric Format (BMF) JSON-Daten zu generieren.

### `--seed <SEED>`

Der Seed für den Pseudozufallszahlengenerator.
Wenn nicht angegeben, wird ein zufälliger Seed verwendet.

### `--count <COUNT>`

Die Anzahl der zu erzeugenden simulierten [Benchmarks][benchmark].

[benchmark]: /de/docs/explanation/benchmarking/#benchmark

### `--measure <MEASURE>`

Die [Measure][measure], die für jeden [Benchmark][benchmark] verwendet werden soll.
Mehrere Measures können angegeben werden, indem die Option `--measure` mehrfach verwendet wird.
Standardmäßig ist die eingebaute `latency`-Measure.

[measure]: /de/docs/explanation/benchmarking/#measure
[benchmark]: /de/docs/explanation/benchmarking/#benchmark

### `--pow <POW>`

Die Zehnerpotenz, die für die simulierten [Metriken][metric] verwendet werden soll.
Standardmäßig ist sie `1`.

[metric]: /de/docs/explanation/benchmarking/#metric

### `--fail`

Beim Ausführen fehlschlagen.

### `--flaky`

Während der Ausführung gelegentlich fehlschlagen.

### `--help`

Hilfemeldung anzeigen.

### `--token <TOKEN>`

<br />

> 🐰 VERALTET: API-Tokens sind zugunsten von API-Schlüsseln veraltet.
> Neue API-Tokens können nicht mehr erstellt werden; bestehende API-Tokens funktionieren weiterhin.
> Verwenden Sie stattdessen [die `--key` Option](#--key-key).

Optional: Entweder die Option `--token` oder die Umgebungsvariable `BENCHER_API_TOKEN` kann auf ein gültiges API-Token gesetzt werden. Wenn beide angegeben sind, hat die Option `--token` Vorrang vor der Umgebungsvariable `BENCHER_API_TOKEN`. Wenn weder die Option `--token` noch [die `--key` Option](#--key-key) angegeben ist, muss das Projekt `unclaimed` sein. Das heißt, wenn Sie bereits ein Projekt `claimed` haben, müssen Sie einen gültigen API-Schlüssel oder ein gültiges API-Token bereitstellen. Steht in Konflikt mit [der `--key` Option](#--key-key).

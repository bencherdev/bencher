### `--ci-id <ID>`

<br />

Необязательно: Пользовательский идентификатор для отправки результатов в CI. По умолчанию Bencher автоматически сегментирует результаты по комбинации: Проект, Ветка, Тестовая среда и [Адаптер](/ru/docs/explanation/adapters/). Установка пользовательского идентификатора полезна, когда Bencher запускается несколько раз в том же CI-потоке для той же комбинации Проекта, Ветки, Тестовой среды и Адаптера. Идентификатор также добавляется к имени проверки GitHub (например, `Bencher Report (<ID>)`), поэтому каждый запуск получает стабильное имя, которое можно использовать как отдельную обязательную проверку статуса. Требуется: `--github-actions`

## `bencher mock`

La sous-commande CLI `bencher mock` est utilisée pour générer des données JSON factices au format Bencher Metric Format (BMF).

### `--seed <SEED>`

La graine pour le générateur de nombres pseudorandomiques.
Si elle n'est pas spécifiée, une graine aléatoire sera utilisée.

### `--count <COUNT>`

Le nombre de [Benchmarks][benchmark] factices à générer.

[benchmark]: /fr/docs/explanation/benchmarking/#benchmark

### `--measure <MEASURE>`

La [Mesure][measure] à utiliser pour chaque [Benchmark][benchmark].
Plusieurs Mesures peuvent être spécifiées en utilisant l'option `--measure` plusieurs fois.
Par défaut, c'est la Mesure intégrée `latency`.

[measure]: /fr/docs/explanation/benchmarking/#measure
[benchmark]: /fr/docs/explanation/benchmarking/#benchmark

### `--pow <POW>`

La puissance de 10 à utiliser pour les [métriques][metric] factices.
Par défaut, c'est `1`.

[metric]: /fr/docs/explanation/benchmarking/#metric

### `--fail`

Échouer pendant l'exécution.

### `--flaky`

Échouer de façon intermittente pendant l'exécution.

### `--help`

Afficher le message d'aide.

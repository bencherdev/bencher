El binario `runner` es el agente que ejecuta los Jobs de benchmark Bare Metal.
Es la referencia detallada para configurar un [Runner Autoalojado][self hosted runners].
El binario proporciona dos subcomandos:

- [`runner up`][runner up]: Inicia el runner, sondeando y ejecutando Jobs de benchmark.
- [`runner run`][runner run]: Descarga una Image y la ejecuta una vez en el host local, para pruebas.

> 🐰 El binario `runner` es distinto de los subcomandos `bencher runner`.
> El binario `runner` opera un host Runner, mientras que
> [`bencher runner`][runners api] gestiona los recursos Runner a través de la API REST.

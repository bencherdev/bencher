## Suivre vos Benchmarks Bare Metal

Vous êtes maintenant prêt à suivre vos résultats de benchmark sur du matériel dédié !
Pour ce faire, vous utiliserez [la sous-commande CLI `bencher run`][bencher run]
avec les options `--project`, `--image` et `--spec`
pour exécuter vos benchmarks sur un Runner Bare Metal et collecter les résultats.

Nous utiliserons la sous-commande [`bencher noise`][bencher noise] comme commande de benchmark.
Elle mesure le bruit environnemental sur le matériel du Runner
et produit des résultats au format [Bencher Metric Format (BMF)][bmf] avec `--format json`.

Nous utiliserons la [Spec][spec] `intel-v1` pour cibler un Runner Bare Metal basé sur Intel.
Consultez la [page Spec][spec] pour une liste complète des Specs disponibles et les détails de leur matériel.
Remplacez `YOUR_PROJECT_SLUG` par le slug du projet des étapes précédentes et exécutez :

<TerminalCommand code="bencher run --project YOUR_PROJECT_SLUG --image YOUR_PROJECT_SLUG:latest --spec intel-v1 bencher noise --format json" />

L'option `--project` sélectionne le projet pour lequel suivre les résultats,
l'option `--image` définit l'image OCI pour l'exécution à distance,
et l'option `--spec` sélectionne la Spec matérielle pour le Runner.
Tout ce qui suit les options de `bencher run` est la commande de benchmark qui s'exécute à l'intérieur du conteneur.

Vous pouvez maintenant visualiser les résultats de chacun de vos benchmarks dans le navigateur.
Cliquez ou copiez-collez les liens depuis `View results`.

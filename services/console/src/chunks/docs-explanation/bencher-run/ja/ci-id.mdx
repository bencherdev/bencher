### `--ci-id <ID>`

<br />

オプション: CIに結果を投稿するためのカスタムID。
デフォルトでは、Bencherは結果をプロジェクト、ブランチ、テストベッド、および[アダプター](/ja/docs/explanation/adapters/)の組み合わせで自動的に区切ります。
同じプロジェクト、ブランチ、テストベッド、アダプターの組み合わせで、同じCIワークフロー内でBencherを複数回実行する場合にカスタムIDを設定することが有用です。
このIDはGitHubチェック名にも追加されるため（例：`Bencher Report (<ID>)`）、
各実行が個別の必須ステータスチェックとして使用できる安定した名前を持つことになります。
必要条件: `--github-actions`

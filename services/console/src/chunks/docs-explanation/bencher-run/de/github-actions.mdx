### `--github-actions <GITHUB_TOKEN>`

<br />

Optional: Setzen Sie das GitHub API-Authentifizierungstoken.
Die bequemste Methode hierfür ist die Verwendung der [GitHub Actions `GITHUB_TOKEN` Umgebungsvariable][github token] (d.h. `--github-actions ${{ secrets.GITHUB_TOKEN }}`).
Wenn diese Option gesetzt ist und `bencher run` in GitHub Actions verwendet wird,
dann werden die Ergebnisse dem Head-Commit als GitHub-Prüfung mit dem Namen `Bencher Report` hinzugefügt (oder `Bencher Report (<ID>)`, wenn `--ci-id` gesetzt ist).
Die GitHub-Prüfung schlägt fehl, wenn ein Alert generiert wird, sodass sie als erforderliche Statusprüfung im Branch-Schutz verwendet werden kann.
Die GitHub-Prüfung wird mit dem Status `in_progress` erstellt, bevor die Benchmarks starten, und wird mit den Ergebnissen abgeschlossen, sobald sie beendet sind.
Wenn `bencher run` mit einem Fehler endet, bevor die Ergebnisse veröffentlicht wurden, wird die GitHub-Prüfung als fehlgeschlagen markiert.
Dies erfordert, dass das Token [den `checks` Bereich mit `write` Berechtigungen][github token permissions] hat.
Wenn die GitHub-Prüfung nicht erstellt werden kann, protokolliert `bencher run` eine Warnung und fährt fort.
Wenn `bencher run` als Teil einer Pull-Anfrage verwendet wird,
dann werden die Ergebnisse auch als Kommentar zur Pull-Anfrage hinzugefügt.
Dies erfordert, dass das Token [den `pull-requests` Bereich mit `write` Berechtigungen][github token permissions] hat.
Die Ergebnisse werden außerdem immer zur Zusammenfassung des Jobs hinzugefügt.

[github token]: https://docs.github.com/de/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/de/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 Wenn Sie innerhalb eines Docker-Containers in einer GitHub-Aktion laufen, müssen Sie die folgenden Umgebungsvariablen übergeben und den Pfad einbinden, der durch `GITHUB_EVENT_PATH` angegeben wird:
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
> - `GITHUB_API_URL` (GitHub Enterprise Server nur)

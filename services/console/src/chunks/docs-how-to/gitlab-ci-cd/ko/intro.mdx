사용 사례에 따라 GitLab CI/CD에서 [지속적인 벤치마킹][continuous benchmarking]을 설정할 수 있습니다:

- [대상 브랜치](#target-branch)
- [병합 요청](#merge-requests)

계속하기 전에 프로젝트 API 키를 생성하고 이를 `BENCHER_API_KEY`이라는 마스크된 변수로 설정해야 합니다! `Your Repo -> Settings -> CI/CD -> Variables -> Expand -> Add variable`로 이동하세요. 변수 키는 `BENCHER_API_KEY`이어야 하고, 변수 값은 귀하의 프로젝트 API 키이어야 합니다. `Protect variable` 및 `Mask variable` 상자 모두를 체크하세요.

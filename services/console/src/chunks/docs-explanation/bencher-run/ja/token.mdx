### `--token <TOKEN>`

<br />

> 🐰 非推奨: API トークンは非推奨となり、API キーに置き換えられました。
> 新しい API トークンは作成できなくなりましたが、既存の API トークンは引き続き使用できます。
> 代わりに[`--key` オプション](#--key-key)を使用してください。

オプション: `--token` オプションまたは `BENCHER_API_TOKEN` 環境変数のいずれかを設定して、有効な API トークンを指定できます。
両方が指定されている場合、`--token` オプションが `BENCHER_API_TOKEN` 環境変数よりも優先されます。
`--token` オプションと[`--key` オプション](#--key-key)のどちらも指定されていない場合、プロジェクトは `unclaimed` でなければなりません。
つまり、すでにプロジェクトを `claimed` している場合、有効な API キーまたは API トークンを提供しなければなりません。
[`--key` オプション](#--key-key)と競合します。

#### Spec
Eine Spec ist eine Hardwarespezifikation, die die verfügbaren Ressourcen einer Testumgebung beschreibt.
Sie umfasst die CPU-Architektur, die Anzahl der CPUs, die Speichergröße, die Festplattengröße und ob die Umgebung Netzwerkzugriff hat.
Specs sind serverweit gültig, sodass dieselbe Spec von mehreren Testumgebungen gemeinsam genutzt werden kann.

Wenn ein Lauf einen Bare Metal Job erstellt, bestimmt die Spec
sowohl die Hardwareressourcen als auch den Standard-Testbed-Namen:

| [`--testbed`](/de/docs/explanation/bencher-run/#--testbed-testbed) angegeben | [`--image`](/de/docs/explanation/bencher-run/#--image-image) angegeben | Quelle des Testbed-Namens    |
|-----------------------|-----------------|------------------------------|
| Nein                  | Nein            | Betriebssystem des Kontexts  |
| Ja                    | Nein            | Expliziter Wert des Benutzers|
| Nein                  | Ja              | Name der aufgelösten Spec (Spec dem Testbed zugewiesen) |
| Ja                    | Ja              | Expliziter Wert des Benutzers (Spec dem Testbed zugewiesen) |

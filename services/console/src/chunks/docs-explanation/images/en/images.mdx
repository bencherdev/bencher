Bare Metal Runners execute benchmarks packaged in [OCI Image Format][oci].

[oci]: https://github.com/opencontainers/image-spec

## Lifecycle

1. User builds and tags an OCI image
```sh
docker build --tag registry.bencher.dev/YOUR_PROJECT_SLUG:my-tag .
```
2. User pushes the OCI image to the Bencher Cloud Registry
```sh
docker push registry.bencher.dev/YOUR_PROJECT_SLUG:my-tag
```
3. User invokes `bencher run` with the OCI image specified with [the `--image` option][image]
```sh
bencher run --image YOUR_PROJECT_SLUG:my-tag
```
4. Bencher API server creates a [Job][job] for the benchmark execution, using the provided Image and [Spec][spec]

[image]: /docs/explanation/bencher-run/#--image-image
[job]: /docs/explanation/bare-metal/#job
[spec]: /docs/explanation/bare-metal/#spec


Then the Bencher API server parses the [benchmark command][benchmark command] output
using a [benchmark harness adapter][adapters].

[benchmark command]: /docs/explanation/bencher-run/#benchmark-command
[adapters]: /docs/explanation/adapters/

## Sandbox

Some [Bare Metal Specs][specs] will require a Sandbox,
so there is a slight, consistent overhead to running your benchmarks.
All Bencher Cloud Specs that run on [Shared Runners][runner], such as `intel-v1` require the Firecracker Sandbox.
To run without a Sandbox, you can opt-in to another [Runner type][runner].

[oci]: https://github.com/opencontainers/image-spec

## Network Access

Some [Bare Metal Specs][specs] will disable network access,
so everything that you need to execute your benchmarks must to be packaged
inside of the OCI image.
All Bencher Cloud Specs that run on [Shared Runners][runner], such as `intel-v1` have network access disabled.
To allow network access, you can opt-in to another [Runner type][runner].

## Build, Tag, and Push the Image

Build your image and push it to the Bencher OCI registry:

```bash
docker build --tag registry.bencher.dev/YOUR_PROJECT_SLUG:my-tag .
docker push registry.bencher.dev/YOUR_PROJECT_SLUG:my-tag
```

Only the OCI registry for the Bencher API server is supported.
For Bencher Self-Hosted, use the local registry:

```bash
docker build -t localhost:61016/my-benchmarks:v1 .
docker push localhost:61016/my-benchmarks:v1
```

## Digest vs Tag

Tags are mutable -- pushing to the same tag overwrites the previous image.
Digests are immutable and uniquely identify an exact image.

When you submit a job with `--image my-image:tag`, Bencher resolves the tag to a digest at submission time.
The job always runs the exact image that was resolved at submission time,
even if the tag is updated afterward.

## Entrypoint and CMD Overrides

You can override the container entrypoint with the [`--entrypoint`][entrypoint] flag on `bencher run`:

```bash
bencher run --image my-image:tag --entrypoint /bin/sh -- -c "./run-benchmarks.sh"
```

Override cmd by passing the benchmark command after `bencher run` options:

```bash
bencher run --image my-image:tag -- cargo bench --features my-feature
```

## Environment Variables

Pass environment variables to the container with [`--env`][env] (can be specified multiple times):

```bash
bencher run --image my-image:tag --env BENCH_SIZE=large --env RUST_LOG=info -- cargo bench
```

## Dockerfile Examples by Adapter

Below is a simple Dockerfile example for each [Bencher adapter][adapters].
Each Dockerfile should:
- Use an appropriate base image for the language
- Install the benchmark harness
- Copy benchmark code
- Set the entrypoint to run the benchmark

### Magic (default)

The [Magic adapter][magic] (`magic`) auto-detects the output format.
Use any of the adapter-specific Dockerfiles below.
For best results, specify an explicit adapter with `--adapter`.

### JSON

The [JSON adapter][json] (`json`) expects [Bencher Metric Format (BMF) JSON][bmf].
This is perfect for custom benchmark harnesses.

```dockerfile
FROM python:3.12-slim
WORKDIR /bench
COPY bench.py .
ENTRYPOINT ["python", "bench.py"]
```

Where `bench.py` outputs BMF JSON to stdout:

```python
import json
import time

start = time.perf_counter_ns()
# ... your benchmark code ...
elapsed_ns = time.perf_counter_ns() - start

print(json.dumps({
    "my_benchmark": {
        "latency": {
            "value": elapsed_ns
        }
    }
}))
```

### Build Time

The [Build Time adapter][build time] measures compile time using the `--build-time` flag on `bencher run`.

```dockerfile
FROM rust:1.87-slim
WORKDIR /build
COPY . .
ENTRYPOINT ["cargo", "build", "--release"]
```

Use with `bencher run --adapter json --build-time --image my-image:tag`.

### File Size

The [File Size adapter][file size] measures file sizes using the `--file-size` option on `bencher run`.

```dockerfile
FROM rust:1.87-slim
WORKDIR /build
COPY . .
RUN cargo build --release
ENTRYPOINT ["ls", "-la", "target/release/my-binary"]
```

Use with `bencher run --adapter json --file-size target/release/my-binary --image my-image:tag`.

### C# BenchmarkDotNet

The [C# DotNet adapter][c sharp dot net] (`c_sharp_dot_net`) expects [BenchmarkDotNet](https://github.com/dotnet/BenchmarkDotNet) output.

```dockerfile
FROM mcr.microsoft.com/dotnet/sdk:9.0
WORKDIR /bench
COPY . .
RUN dotnet restore
ENTRYPOINT ["dotnet", "run", "-c", "Release", "--exporters", "json"]
```

### C++ Catch2

The [C++ Catch2 adapter][cpp catch2] (`cpp_catch2`) expects [Catch2](https://github.com/catchorg/Catch2) output.

```dockerfile
FROM gcc:14
WORKDIR /bench
COPY . .
RUN make bench
ENTRYPOINT ["./bench"]
```

### C++ Google Benchmark

The [C++ Google adapter][cpp google] (`cpp_google`) expects [Google Benchmark](https://github.com/google/benchmark) output in JSON format.

```dockerfile
FROM gcc:14
RUN apt-get update && apt-get install -y cmake libbenchmark-dev
WORKDIR /bench
COPY . .
RUN cmake -B build -DCMAKE_BUILD_TYPE=Release && cmake --build build
ENTRYPOINT ["./build/my_benchmark", "--benchmark_format=json"]
```

### Dart Benchmark Harness

The [Dart Benchmark Harness adapter][dart benchmark harness] (`dart_benchmark_harness`) expects [benchmark_harness](https://pub.dev/packages/benchmark_harness) output.

```dockerfile
FROM dart:stable
WORKDIR /bench
COPY . .
RUN dart pub get
ENTRYPOINT ["dart", "run", "bench.dart"]
```

### Go Bench

The [Go Bench adapter][go bench] (`go_bench`) expects `go test -bench` output.

```dockerfile
FROM golang:1.24
WORKDIR /bench
COPY . .
RUN go mod download
ENTRYPOINT ["go", "test", "-bench=.", "-benchmem", "./..."]
```

### Java JMH

The [Java JMH adapter][java jmh] (`java_jmh`) expects [JMH](https://github.com/openjdk/jmh) output in JSON format.

```dockerfile
FROM maven:3.9-eclipse-temurin-21
WORKDIR /bench
COPY . .
RUN mvn clean package -DskipTests
ENTRYPOINT ["java", "-jar", "target/benchmarks.jar", "-rf", "json"]
```

### JavaScript Benchmark.js

The [JavaScript Benchmark adapter][js benchmark] (`js_benchmark`) expects [Benchmark.js](https://github.com/bestiejs/benchmark.js) output.

```dockerfile
FROM node:22-slim
WORKDIR /bench
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
ENTRYPOINT ["node", "bench.js"]
```

### JavaScript console.time

The [JavaScript Time adapter][js time] (`js_time`) expects `console.time`/`console.timeEnd` output.

```dockerfile
FROM node:22-slim
WORKDIR /bench
COPY . .
ENTRYPOINT ["node", "bench.js"]
```

### Python ASV

The [Python ASV adapter][python asv] (`python_asv`) expects [airspeed velocity](https://github.com/airspeed-velocity/asv) output.

```dockerfile
FROM python:3.12-slim
WORKDIR /bench
COPY . .
RUN pip install asv virtualenv
ENTRYPOINT ["asv", "run"]
```

### Python pytest-benchmark

The [Python Pytest adapter][python pytest] (`python_pytest`) expects [pytest-benchmark](https://github.com/ionelmc/pytest-benchmark) output in JSON format.

```dockerfile
FROM python:3.12-slim
WORKDIR /bench
COPY . .
RUN pip install pytest pytest-benchmark
ENTRYPOINT ["pytest", "--benchmark-json=results.json"]
```

### Ruby Benchmark

The [Ruby Benchmark adapter][ruby benchmark] (`ruby_benchmark`) expects [Benchmark module](https://github.com/ruby/benchmark) output.

```dockerfile
FROM ruby:3.3-slim
WORKDIR /bench
COPY . .
RUN bundle install
ENTRYPOINT ["ruby", "bench.rb"]
```

### Rust libtest bench

The [Rust Bench adapter][rust bench] (`rust_bench`) expects `cargo bench` with [libtest bench](https://doc.rust-lang.org/rustc/tests/index.html#benchmarks) output.

```dockerfile
FROM rust:1.87-slim
RUN rustup default nightly
WORKDIR /bench
COPY . .
ENTRYPOINT ["cargo", "bench"]
```

### Rust Criterion

The [Rust Criterion adapter][rust criterion] (`rust_criterion`) expects [Criterion](https://github.com/bheisler/criterion.rs) output.

```dockerfile
FROM rust:1.87-slim
WORKDIR /bench
COPY . .
ENTRYPOINT ["cargo", "bench"]
```

### Rust Iai

The [Rust Iai adapter][rust iai] (`rust_iai`) expects [Iai](https://github.com/bheisler/iai) output.

```dockerfile
FROM rust:1.87-slim
RUN apt-get update && apt-get install -y valgrind
WORKDIR /bench
COPY . .
ENTRYPOINT ["cargo", "bench"]
```

### Rust Gungraun

The [Rust Gungraun adapter][rust gungraun] (`rust_gungraun`) expects [Gungraun](https://github.com/gungraun/gungraun) output.

```dockerfile
FROM rust:1.87-slim
RUN apt-get update && apt-get install -y valgrind
WORKDIR /bench
COPY . .
ENTRYPOINT ["cargo", "bench"]
```

Iai-Callgrind has been renamed to [Gungraun][rust gungraun].

### Shell Hyperfine

The [Shell Hyperfine adapter][shell hyperfine] (`shell_hyperfine`) expects [Hyperfine](https://github.com/sharkdp/hyperfine) output in JSON format.

```dockerfile

WORKDIR /bench
COPY . .
```

## Best Practices

- **Keep images small**: Use multi-stage builds to reduce image size and attack surface.
- **Pin dependencies**: Lock dependency versions for reproducible benchmarks.
- **Stdout for results only**: Output only benchmark results to stdout. Use stderr for logs and diagnostics.
- **Test locally first**: Verify your image works before pushing:

```bash
docker run my-image:tag
```

## Spec

Une Spec est une spécification matérielle qui décrit les ressources disponibles pour un [Runner][runner] :

1. Système d'exploitation (OS)
   1. Linux
   2. macOS
   3. Windows
2. Architecture CPU
   1. x86_64
   2. ARM64
3. Sandbox
   1. Firecracker microVM
   2. Pas de Sandbox
4. Nombre de CPU
5. Taille de la mémoire (RAM)
6. Taille du disque
7. Accès réseau

Les Specs ont une portée serveur ou organisation,
de sorte que la même Spec peut être partagée entre plusieurs [Testbeds][testbed] sur le même serveur ou au sein de la même organisation respectivement.

L'ordre de priorité pour la sélection de la Spec lors de l'utilisation de <code><a href="/docs/explanation/bencher-run/">bencher run</a></code> est le suivant :
1. L'option `--spec`
2. La Spec assignée au Testbed
3. La Spec par défaut au niveau de l'organisation
4. La Spec par défaut au niveau du serveur

La Spec sélectionnée sera alors assignée au [Testbed][testbed] courant.

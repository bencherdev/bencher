### `--spec <SPEC>`

<br />

Spec의 슬러그 또는 UUID를 지정합니다.
Spec은 [Runner][runner]에서 사용할 수 있는 리소스를 설명하는 하드웨어 사양입니다:

1. 운영 체제 (OS)
   1. Linux
   2. macOS
   3. Windows
2. CPU 아키텍처
   1. x86_64
   2. ARM64
3. Sandbox
   1. Firecracker microVM
   2. Sandbox 없음
4. CPU 수
5. 메모리 크기 (RAM)
6. 디스크 크기
7. 네트워크 접근

현재 [Bencher Cloud][bencher cloud]에서 다음 Spec을 사용할 수 있습니다:

|   이름   |   슬러그   |  OS   | 아키텍처 |       Sandbox       | CPU |  메모리  |  디스크   | 네트워크 |
| :------: | :--------: | :---: | :------: | :-----------------: | :-: | :------: | :-------: | :------: |
| Intel v1 | `intel-v1` | Linux |  x86_64  | Firecracker microVM |  4  | 48.0 GiB | 128.0 GiB |   없음   |

Spec은 서버 범위 또는 조직 범위를 가지므로,
동일한 Spec을 같은 서버의 여러 테스트베드 또는 같은 조직 내에서 공유할 수 있습니다.

<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code>을 사용할 때 Spec 선택 우선순위는 다음과 같습니다:
1. `--spec` 옵션
2. 테스트베드에 할당된 Spec
3. 기본 조직 범위 Spec
4. 기본 서버 범위 Spec

선택된 Spec은 현재 [테스트베드][testbed]에 할당됩니다.
이 옵션을 사용하려면 [`--image` 옵션][image]이 설정되어 있어야 합니다.

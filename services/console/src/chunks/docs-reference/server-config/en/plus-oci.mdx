### `plus.oci`

This section specifies the OCI (Open Container Initiative) registry storage configuration.
Bencher can function as an OCI-compliant container/artifact registry.

If this section is not configured, Bencher will use local filesystem storage
in an `oci` directory sibling to the database file.
This is suitable for development and single-instance deployments.

For production deployments with multiple instances or for durability,
configure S3-based storage.

There is one storage `service`:
- `aws_s3`: Store OCI artifacts in AWS S3
    - `access_key_id`: AWS access key ID
    - `secret_access_key`: AWS secret access key
    - `access_point`: S3 Access Point ARN with optional path prefix.
      Format: `arn:aws:s3:<region>:<account-id>:accesspoint/<access-point-name>[/<path-prefix>]`

|             Name             |                               Example                               | Default | Required |                                       Description                                        |
| :--------------------------: | :-----------------------------------------------------------------: | :-----: | :------: | :--------------------------------------------------------------------------------------: |
|          data_store          |                              \{ ... \}                              |   ---   |    No    |               Specifies the OCI storage backend. Local if omitted.                       |
|      data_store.service      |                              "aws_s3"                               |   ---   |   Yes    |                       Specifies the storage service type.                                |
|   data_store.access_key_id   |                       "AKIAIOSFODNN7EXAMPLE"                        |   ---   |   Yes    |                                AWS access key ID.                                        |
| data_store.secret_access_key |             "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"              |   ---   |   Yes    |                              AWS secret access key.                                      |
|   data_store.access_point    | "arn:aws:s3:us-east-1:123456789012:accesspoint/my-access-point/oci" |   ---   |   Yes    |        S3 Access Point ARN. Optional path prefix after access point name.                |
|       upload_timeout         |                                3600                                 |  3600   |    No    | Upload session timeout in seconds. Stale uploads are cleaned up when new uploads start. |

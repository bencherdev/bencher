## `bencher mock`

Подкоманда CLI `bencher mock` используется для генерации макетных JSON-данных в формате Bencher Metric Format (BMF).

### `--seed <SEED>`

Начальное значение (seed) для генератора псевдослучайных чис.
Если не указано, будет использовано случайное значение.

### `--count <COUNT>`

Количество макетных [бенчмарков][benchmark], которые нужно сгенерировать.

[benchmark]: /ru/docs/explanation/benchmarking/#benchmark

### `--measure <MEASURE>`

Какую [меру][measure] использовать для каждого [бенчмарка][benchmark].
Можно указать несколько мер, используя опцию `--measure` несколько раз.
По умолчанию используется встроенная мера `latency`.

[measure]: /ru/docs/explanation/benchmarking/#measure
[benchmark]: /ru/docs/explanation/benchmarking/#benchmark

### `--pow <POW>`

Степень десяти, используемая для макетных [метрик][metric].
По умолчанию — `1`.

[metric]: /ru/docs/explanation/benchmarking/#metric

### `--fail`

Завершаться с ошибкой во время выполнения.

### `--flaky`

Периодически давать сбой во время выполнения.

### `--help`

Вывести справку.

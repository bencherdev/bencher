Le binaire `runner` est l'agent qui exécute les Jobs de benchmark Bare Metal.
Il constitue la référence détaillée pour la mise en place d'un [Runner auto-hébergé][self hosted runners].
Le binaire fournit deux sous-commandes :

- [`runner up`][runner up] : Démarre le runner, en interrogeant et en exécutant les Jobs de benchmark.
- [`runner run`][runner run] : Récupère une Image et l'exécute une seule fois sur l'hôte local, à des fins de test.

> 🐰 Le binaire `runner` est distinct des sous-commandes `bencher runner`.
> Le binaire `runner` exploite un hôte Runner, tandis que
> [`bencher runner`][runners api] gère les ressources Runner via l'API REST.

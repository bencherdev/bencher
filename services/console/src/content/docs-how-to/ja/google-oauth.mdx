Bencher Cloud をご利用の場合は、`Google` ボタンで簡単に [サインアップ](/auth/signup) または [ログイン](/auth/login) できます。

Bencher Self-Hosted を利用する場合は、[Bencher Plus Enterprise プラン][bencher plus enterprise] が必要で、独自の Google OAuth2 クライアントを作成し、[コンソールサーバーの設定][console config] と [API サーバーの設定][server config] を更新する必要があります。

[bencher plus enterprise]: /pricing/?level=enterprise
[console config]: /ja/docs/reference/console-config/
[server config]: /ja/docs/reference/server-config/

## Google OAuth 2.0 クライアントの作成

- [Google API Console][google api console] にアクセスする
- 左サイドバーの [ `Credentials` ページ][google cloud credentials] に移動する
- `OAuth 2.0 Client ID` を作成する
  - `➕ Create Credentials` をクリック > `OAuth client ID`
  - 同意画面を設定する
    - Get started
      - App Information
        - App name
          - 名前を選択する（例: `Bencher for My Enterprise`）
        - User support email
          - メールアドレスを選択する
      - Audience
        - `Internal`
          - 組織内のユーザーのみ利用可能です。アプリの検証を提出する必要はありません。
      - Contact Information
        - メールアドレスを追加する
          - これらのメールアドレスは、プロジェクトの変更について Google が通知するためのものです。
      - Finish
        - Google API Services User Data Policy（Google API サービスのユーザーデータポリシー）に同意する
  - `Create OAuth Client` をクリック
    - Application type
      - `Desktop app` を選択
      - Name
        - 名前を選択する（例: `Bencher for My Enterprise`）
      - `Create` をクリック
  - OAuth Client が作成される
    - `Client ID` をメモする（例: `012345-abcd6789envs.apps.googleusercontent.com`）
    - `OK` をクリック
    - 新しく作成した OAuth 2.0 Client ID をクリック
    - `Client Secret` をメモする（例: `GOCSPX-abcdefgh1234567890wxyz`）

[google api console]: https://console.developers.google.com/
[google cloud credentials]: https://console.cloud.google.com/apis/credentials

> 🐰 6 か月以上使用されていない Google OAuth クライアントは削除される可能性があります。
> 削除に関する通知は Google から送られ、削除後 30 日以内であればクライアントを復元できます。

## Google OAuth クライアントの設定

### コンソール UI
`bencher-console` の Docker イメージを実行する際に、環境変数 `OAUTH_GOOGLE` を `true` に設定します。（例: `OAUTH_GOOGLE=true`）
Client ID が設定されていないと、UI に Google ボタンは表示されません。

### API サーバー
`client_id` と `client_secret` を [サーバー設定の `plus.google` セクション](/ja/docs/reference/server-config/#plusgoogle) に追加してください。
Google セクションが存在しない場合、サーバーは Google 認証のリクエストを処理できず失敗します。

### `--github-actions <GITHUB_TOKEN>`

<br />

Optional: Set the GitHub API authentication token.
The most convenient way to do this is the [GitHub Actions `GITHUB_TOKEN` environment variable][github token] (ie `--github-actions ${{ secrets.GITHUB_TOKEN }}`).
When this option is set and `bencher run` is used in GitHub Actions,
then the results will be added to the head commit as a GitHub Check named `Bencher Report` (or `Bencher Report (<ID>)` if `--ci-id` is set).
The GitHub Check will fail if an Alert is generated, so it can be used as a required status check in branch protection.
The GitHub Check is created with an `in_progress` status before the benchmarks start and is completed with the results once they finish.
If `bencher run` exits with an error before the results are posted, then the GitHub Check is marked as failed.
This requires the token to have [the `checks` scope with `write` permissions][github token permissions].
If the GitHub Check cannot be created, then `bencher run` will log a warning and continue.
When `bencher run` is used as a part of a pull request,
then the results will also be added to the pull request as a comment.
This requires the token to have [the `pull-requests` scope with `write` permissions][github token permissions].
The results will always be added to the job summary as well.

[github token]: https://docs.github.com/en/actions/security-guides/automatic-token-authentication
[github token permissions]: https://docs.github.com/en/actions/security-for-github-actions/security-guides/automatic-token-authentication#permissions-for-the-github_token

> 🐰 If you are running inside of a Docker container within GitHub Action, you will need to pass in the following environment variables and mount the path specified by `GITHUB_EVENT_PATH`:
> - `GITHUB_ACTIONS`
> - `GITHUB_EVENT_NAME`
> - `GITHUB_EVENT_PATH`
> - `GITHUB_SHA`
> - `GITHUB_API_URL` (GitHub Enterprise Server only)

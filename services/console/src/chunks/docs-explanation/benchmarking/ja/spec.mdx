#### Spec
Specは、テストベッドで利用可能なリソースを記述するハードウェア仕様です。
CPUアーキテクチャ、CPU数、メモリサイズ、ディスクサイズ、および環境がネットワークアクセスを持つかどうかが含まれます。
Specはサーバースコープであるため、同じSpecを複数のテストベッドで共有できます。

実行がBare Metal Jobを作成する場合、Specは
ハードウェアリソースとデフォルトのTestbed名の両方を決定します：

| [`--testbed`](/ja/docs/explanation/bencher-run/#--testbed-testbed) 指定 | [`--image`](/ja/docs/explanation/bencher-run/#--image-image) 指定 | Testbed名のソース         |
|-----------------|---------------|--------------------------|
| なし             | なし           | コンテキストのOS          |
| あり             | なし           | ユーザーの明示的な値      |
| なし             | あり           | 解決されたSpecの名前（SpecがTestbedに割り当てられる） |
| あり             | あり           | ユーザーの明示的な値（SpecがTestbedに割り当てられる） |

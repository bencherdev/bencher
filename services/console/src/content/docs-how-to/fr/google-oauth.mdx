Si vous utilisez Bencher Cloud, vous pouvez simplement [vous inscrire](/auth/signup) ou [vous connecter](/auth/login) en utilisant le bouton `Google`.

Pour Bencher auto‑hébergé, vous devez disposer d'un [plan Bencher Plus Enterprise][bencher plus enterprise], créer votre propre client Google OAuth2, et mettre à jour votre [configuration du serveur de la console][console config] et la [configuration du serveur API][server config].

[console config]: /fr/docs/reference/console-config/
[server config]: /fr/docs/reference/server-config/

## Créer un client Google OAuth 2.0

- Accédez à la [Google API Console][google api console]
- Allez sur la [page `Credentials`][google cloud credentials] dans la barre latérale gauche
- Créez un `OAuth 2.0 Client ID`
  - Cliquez sur `➕ Create Credentials` > `OAuth client ID`
  - Configurez l'écran de consentement
    - Démarrer
      - Informations sur l'application
        - Nom de l'application
          - Choisissez un nom (ex : `Bencher for My Enterprise`)
        - E‑mail d'assistance utilisateur
          - Choisissez une adresse e‑mail
      - Public cible
        - `Internal`
          - Disponible uniquement pour les utilisateurs de votre organisation.
            Vous n'aurez pas besoin de soumettre votre application pour vérification.
      - Informations de contact
        - Ajoutez votre adresse e‑mail
          - Ces adresses e‑mail permettent à Google de vous notifier de tout changement concernant votre projet.
      - Terminer
        - Acceptez la Politique relative aux données des utilisateurs des services Google API
  - Cliquez sur `Create OAuth Client`
    - Type d'application
      - Sélectionnez `Desktop app`
      - Nom
        - Choisissez un nom (ex : `Bencher for My Enterprise`)
      - Cliquez sur `Create`
  - Client OAuth créé
    - Notez le `Client ID` (ex : `012345-abcd6789envs.apps.googleusercontent.com`)
    - Cliquez sur `OK`
    - Cliquez sur le nouvel identifiant OAuth 2.0 créé
    - Notez le `Client Secret` (ex : `GOCSPX-abcdefgh1234567890wxyz`)

[google api console]: https://console.developers.google.com/
[google cloud credentials]: https://console.cloud.google.com/apis/credentials

> 🐰 Les clients Google OAuth inactifs peuvent être supprimés s'ils ne sont pas utilisés pendant 6 mois.
> Google vous notifiera de la suppression pour inactivité et vous pourrez restaurer les clients jusqu'à 30 jours après la suppression.

## Configuration du client Google OAuth

### Interface de la console
Définissez la variable d'environnement `OAUTH_GOOGLE` à `true` lorsque vous lancez l'image Docker `bencher-console`. (ex : `OAUTH_GOOGLE=true`)
Si le Client ID est manquant, les boutons Google n'apparaîtront pas dans l'interface.

### Serveur API
Ajoutez `client_id` et `client_secret` à [la section `plus.google` de la configuration du serveur](/fr/docs/reference/server-config/#plusgoogle).
Si la section Google est absente, le serveur retournera une erreur pour toutes les requêtes d'authentification Google.

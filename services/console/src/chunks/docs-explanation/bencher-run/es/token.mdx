### `--token <TOKEN>`

<br />

> 🐰 OBSOLETO: Los tokens de API están obsoletos en favor de las claves de API.
> Ya no se pueden crear nuevos tokens de API; los tokens de API existentes siguen funcionando.
> Use [la opción `--key`](#--key-key) en su lugar.

Opcional: Puede establecerse la opción `--token` o la variable de entorno `BENCHER_API_TOKEN` a un token de API válido.
Si ambos están especificados, la opción `--token` tiene prioridad sobre la variable de entorno `BENCHER_API_TOKEN`.
Si no se especifica ni la opción `--token` ni [la opción `--key`](#--key-key), el Proyecto debe estar `no reclamado`.
Es decir, si ya ha `reclamado` un Proyecto, debe proporcionar una clave de API o un token de API válidos.
Entra en conflicto con [la opción `--key`](#--key-key).

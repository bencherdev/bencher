#### Spec
Uma Spec é uma especificação de hardware que descreve os recursos disponíveis para um Testbed.
Inclui a arquitetura de CPU, número de CPUs, tamanho de memória, tamanho de disco e se o ambiente tem acesso à rede.
As Specs têm escopo de servidor, portanto a mesma Spec pode ser compartilhada entre múltiplos Testbeds.

Quando uma execução cria um Bare Metal Job, a Spec determina
tanto os recursos de hardware quanto o nome padrão do Testbed:

| `--testbed` fornecido | Job solicitado | Fonte do nome do Testbed       |
|----------------------|----------------|--------------------------------|
| Não                  | Não            | SO do contexto                 |
| Sim                  | Não            | Valor explícito do usuário     |
| Não                  | Sim            | Nome da Spec resolvida (Spec atribuída ao Testbed) |
| Sim                  | Sim            | Valor explícito do usuário (Spec atribuída ao Testbed) |
